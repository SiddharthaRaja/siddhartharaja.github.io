Multimodal Breast Cancer Subtype Classifier GitHub ↗
Jan 2026 – Present
Predicting breast cancer IHC subtypes from genomics + histology without receptor testing. Identified and resolved fundamental feature leakage inflating accuracy to 86% — rebuilt honest pipeline achieving ROC-AUC 0.76.
PyTorch XGBoost HPC SLURM H100 OpenSlide
details
Key Results
ROC-AUC 0.76 on honest pipeline after removing feature leakage
Circular Logic → Corrected inflated accuracy identified and eliminated via data audit
4 Subtypes HER2+, HR+, TNBC, HR+/HER2+ classified from genomic + histology fusion
Approach
Combined RNA-seq gene expression features with whole-slide image (WSI) patch embeddings extracted via OpenSlide. Fused modalities using late-fusion XGBoost ensemble trained on TCGA-BRCA cohort. Ran on BITS HPC cluster using SLURM job scheduling across NVIDIA H100 nodes. Emphasis throughout on rigorous train/val/test splits with no label leakage.
What I Learned
Discovered that naive feature concatenation from clinical metadata caused target leakage — a subtle bug that inflated metrics significantly. Rebuilt the entire preprocessing pipeline from scratch with strict data isolation. This project fundamentally shaped how I think about evaluation integrity in ML.

Adversarial LLM Benchmarking Framework GitHub ↗
Apr – May 2026
GPU-accelerated robustness evaluation of open-source LLMs across 7 adversarial attack categories. TinyLlama: 75% jailbreak susceptibility vs Mistral's 6% harmful compliance rate.
AI Safety HuggingFace NVIDIA H100 Streamlit Plotly
details
Key Results
75% susceptibility TinyLlama jailbreak rate across adversarial prompt categories
6% compliance Mistral-7B harmful response rate — strongest open-source performer
7 attack types prompt injection, role-play bypass, jailbreak templates, and more
Approach
Built a modular evaluation harness in Python that runs batched adversarial prompts against HuggingFace-hosted models on H100 GPUs. Defined a taxonomy of 7 attack categories including direct jailbreaks, indirect injections, and role-play bypasses. Results visualised in an interactive Streamlit dashboard with per-model, per-category breakdowns.
What I Learned
Model scale alone doesn't determine safety — alignment training methodology matters far more. Smaller models with RLHF (Mistral) significantly outperformed larger models without it. This work deepened my interest in AI safety evaluation as a discipline distinct from capability benchmarking.

Retrieval-Augmented Generation System for Scientific Literature GitHub ↗
Feb – Mar 2026
End-to-end retrieval-augmented generation pipeline with hybrid retrieval, query expansion, and citation grounding to minimize hallucinations across large document collections.
RAG TF-IDF Semantic Chunking NLP
details
Key Results
Hybrid retrieval TF-IDF + dense embedding fusion outperformed either method alone
Citation grounding every generated answer linked to source paragraph with confidence score
Query expansion HyDE-inspired rewriting improved recall on ambiguous scientific queries
Approach
Ingested PDFs via semantic chunking (respecting section boundaries), indexed with both TF-IDF and sentence-transformer embeddings. At query time, retrieved top-k chunks from both indices and re-ranked with cross-encoder. Generation via a locally-hosted LLM with strict citation constraints — the model was prompted to refuse answers it couldn't ground.
What I Learned
Hallucination reduction is primarily a retrieval problem, not a generation problem. The quality of chunking and retrieval had far more impact than prompt engineering the LLM. Hybrid retrieval consistently beat pure dense or sparse approaches on scientific text.

Explicit Content Text Classifier GitHub ↗
Aug – Dec 2024
Linear SVM with full NLP preprocessing achieving 0.86 precision and 0.96 recall. 3-tier confidence thresholding with a NEEDS REVIEW band for safety-critical content moderation pipelines.
scikit-learn spaCy NLTK NLP
details
Key Results
0.86 precision very low false positive rate — critical for user-facing moderation
0.96 recall near-complete coverage of explicit content in test corpus
3-tier system SAFE / NEEDS REVIEW / BLOCK — human-in-the-loop on uncertain cases
Approach
Built full NLP preprocessing pipeline: tokenisation, lemmatisation (spaCy), stopword removal, n-gram feature extraction. Trained Linear SVM on TF-IDF features. The key design decision was the 3-tier confidence band — instead of binary classification, uncertain predictions in the middle confidence range are flagged for human review, reducing both false positives and false negatives in production.
What I Learned
For safety-critical classifiers, optimising for a single metric is wrong. Precision and recall trade off differently depending on the harm model. A simple linear classifier with careful thresholding and human escalation outperforms a complex model with a single decision boundary.
